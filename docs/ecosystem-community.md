## Getting Help

### Before Asking for Help

1. **Search existing resources**:
   - Project documentation
   - GitHub issues and discussions
   - Discord channel history
   - Universal Blue Discourse

2. **Gather system information**:

   ```bash
   # System information
   rpm-ostree status

   # Hardware information
   inxi -Fxz

   # Container information (if relevant)
   podman version
   podman info
   ```

### Where to Ask Questions

**Quick Questions**: Discord channels for immediate community help

**Technical Issues**: GitHub Issues with:

- Clear problem description
- Steps to reproduce
- System information
- Expected vs actual behavior

**Feature Requests**: GitHub Discussions for community input before formal requests

**General Discussion**: Discord or Discourse for broader community engagement

### Help Etiquette

**Be Specific**: Provide context, error messages, and system details

**Be Patient**: Community members volunteer their time

**Give Back**: Help others when you can, share solutions you find

**Stay On Topic**: Use appropriate channels for your questions

## Contributing

### Ways to Contribute

**Documentation**:

- Improve existing documentation
- Write tutorials and guides
- Translate content
- Report documentation issues

**Code Contributions**:

- Bug fixes and improvements
- New features (discuss first)
- Testing and QA
- Performance optimizations

**Community Support**:

- Help other users in Discord
- Answer questions in GitHub Discussions
- Share your experiences and setups
- Create content (blogs, videos, tutorials)

**Testing and Feedback**:

- Test development builds
- Report bugs and issues
- Provide feedback on new features
- Test on different hardware configurations

### Contribution Guidelines

**For All Projects**:

1. Read the project's CONTRIBUTING.md file
2. Follow the Code of Conduct
3. Search existing issues before creating new ones
4. Use clear, descriptive titles and descriptions
5. Test your changes thoroughly

**Development Setup**:
Most Universal Blue projects use similar development workflows:

```bash
# Fork the repository on GitHub
# Clone your fork
git clone https://github.com/your-username/project-name
cd project-name

# Create a feature branch
git checkout -b feature-description

# Make changes and test
# Commit with clear messages
git commit -m "Add: clear description of changes"

# Push and create pull request
git push origin feature-description
```

## Code of Conduct

All Universal Blue projects follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/). Key points:

- **Be welcoming and inclusive** to all community members
- **Be respectful** in discussions and interactions
- **Be collaborative** and help build a positive community
- **Focus on the project and community good** over individual interests

Report any Code of Conduct violations to project maintainers via:

- Discord direct messages to moderators
- Email to project maintainers
- GitHub repository issue (if appropriate)
